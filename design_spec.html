진행 유력 매치 표시 시스템

매니저가 매치 리스트에서 "진행 가능성이 높은 매치"를 한눈에 알아보고 선택할 수 있도록, 매치 상태를 직관적인 마크로 표시하는 디자인 체계입니다.

최종 업데이트 2026.06 · 매니저 앱 매치 리스트 / 매치 상세

1개요

매치 리스트는 정보 밀도가 높고 카드 폭이 좁습니다(약 158px). 이 제약 안에서 매니저가 스캔만으로 매치의 진행 상태·거래 조건을 파악하도록, 학습이 필요 없는 작은 마크로 신호를 전달합니다.

  • 목표 · 진행률 높은 매치로 매니저 선택을 자연스럽게 유도해 매치 취소율을 낮춤
  • 원칙 · 색이 들어가면 "매치 상태", 무채색이면 "거래 정보"로 읽히게 분리
  • 제약 · 카드 박스 크기는 실제 앱과 동일하게 유지. 마크는 기존 요소를 밀어내지 않는 최소 크기

2매치 상태 마크 체계

상태 신호는 색·형태로, 거래 정보는 무채색으로. 각 마크는 아래와 같습니다.

마크의미 · 표시 방식
▸ 우측 상단 · 시간 줄 — 거래 정보 (무채색)
+5,000원프로모션 금액
추가 정산 금액. 거래 정보라 무채색 그레이.
▸ 우측 하단 · 포맷 줄 — 매치 상태 (색, 상호배타·1개)
진행 유력
최근 5주간 진행이 많이 된 매치. 앱 헤리티지인 주황 화살표를 그대로 사용. 강/약은 꼬리 라인 수로 차등(강=꼬리 2 / 약=꼬리 1).
N신규
최근 새로 열린 매치. 노란 "N" 미니 라벨.
확정확정
최소 인원이 모여 진행이 확정된 매치. 라이트블루 칩.
긴급양도는 메인 리스트에서 제거하고 매치 상세 페이지에서만 노출합니다. MY 배지도 메인 리스트에서 제거했습니다(내 매치는 이미 인지하는 정보). · 강/약 색 농도 차등은 흰 배경에서 흐린 주황이 묻혀 폐기 → 같은 진한 주황을 유지하고 꼬리 라인 수(강 2 / 약 1)로 강도를 표현.

3배치 원칙

한 카드의 우측은 두 구역으로 나뉩니다. 구역이 곧 의미입니다.

우측 상단 · 시간 줄 — 거래

우측 하단 · 포맷 줄 — 매치 상태
매치의 진행 상태 신호(상호배타, 1개)

N 신규
확정 확정

4상호배타 · 우선순위

  • 상태는 상호배타 · 진행 유력 / 신규 / 확정은 한 카드에 최대 1개(포맷 줄 우측)
  • 거래 정보는 병기 가능 · 가격은 상태 마크와 함께 표시(예: 진행 유력 + 5,000원)
  • 긴급양도 · 메인 리스트에서는 미노출, 매치 상세 페이지에서만 안내
  • MY · 메인 리스트 배지에서 제거(내 매치는 이미 인지하고 있는 정보)

6컴포넌트

① 매치 상태 표시 안내 모달

리스트 필터 아래 "매치 상태 표시 안내 ⓘ"로 진입. 진행 유력 · 신규 · 확정 마크의 의미를 설명합니다. 신규 라벨을 처음 본 사용자에게 1회 자동 노출되며(이후 미노출), 바텀시트 형태로 슬라이드업됩니다.

② 정렬 옵션 — 리스트 정렬 순서 변경

"정렬 ▾" 칩 → 바텀시트로 4개 옵션 제공. 옵션을 선택하면 리스트의 정렬 순서가 바뀌어 해당 조건의 매치만 추려 시간순으로 재정렬되고, 상단에 "기준 · N건" 헤더가 표시됩니다. "기본"을 선택하면 날짜별 그룹 정렬로 돌아갑니다.

기본 (날짜순) 시작 임박 · 긴급양도/프로모션 확정 매치 배지 있는 매치

③ 매치 상세 페이지 — 배지 대신 섹션

상세 페이지에서는 진행 유력·신규 배지를 노출하지 않고, 대신 설명 섹션으로 안내합니다. 정보를 읽는 맥락(상세)에서는 충분한 설명이 더 적합하기 때문입니다.

  • 진행 유력 매치 섹션 · 주황 화살표 + "최근 5주 동안 진행이 많이 된 매치예요."
  • 신규 매치 섹션 · 노란 N + "최근에 새로 열린 매치예요. 아직 신청 인원이 많지 않을 수 있어요."

7주요 의사결정 기록

진행 유력 표시를 기존 모양에서 바꾸지 않는 이유

내부 진행률 기준이 바뀌어도(예: 80% → 90%) 사용자가 직접 인지하는 변화가 아니므로, 의미가 동일하다면 시각적 표현은 유지하는 것이 사용자 기대·일관성에 유리합니다. 두 가지 UX 원칙에 근거합니다.

1) Jakob's Law — 사용자의 기존 학습 존중

사용자는 새로운 시스템을 접할 때 서비스의 의도보다 자신의 기존 경험과 학습된 패턴을 기준으로 해석합니다. 매니저는 주황 화살표 = 진행 가능성이 높은 매치로 이미 학습된 상태이며, 내부 기준 변경(80%→90%)은 사용자가 직접 인지하지 못하는 변화입니다. 의미가 동일하다면 기존 시각적 표현을 유지하는 것이 사용자의 기대와 일관성을 유지하는 데 유리합니다.

2) Recognition over Recall — 재인식 우선 원칙

Nielsen Heuristics의 핵심 원칙 중 하나로, 사용자가 새로운 규칙을 기억하도록 만드는 것보다 이미 알고 있는 패턴을 재인식하도록 설계하는 것이 바람직합니다. 기존 표시를 유지하면 사용자는 즉시 의미를 이해할 수 있고, 표시를 변경하면 새로운 의미와 규칙을 다시 학습해야 합니다. 동일한 개념은 동일한 시각적 표현으로 유지하는 것이 인지 부담을 줄입니다.

다만 신규(N)처럼 새로 도입한 마크는 학습이 필요하므로, "매치 상태 표시 안내" 모달(신규 사용자에게 1회 자동 노출)로 의미 인지를 보완했습니다. 즉, 이미 학습된 표시는 그대로 유지하고, 새 표시만 모달로 학습을 돕는 방식입니다.

강/약을 색 농도로 주지 않은 이유?

흐린 주황은 흰 배경에서 묻혀 가시성이 떨어집니다. 같은 진한 주황을 유지하고 꼬리 라인 수(2/1)로 강도를 표현해, "기세가 셀수록 선이 많다"는 직관과 가시성을 동시에 확보.

신규를 노란 N으로 한 이유?

"NEW=노랑/골드"는 가장 널리 통하는 직관. 한 글자 "N"으로 폭을 최소화하고, 팔레트 내 옐로(#FFE875)+앰버(#97500F)로 가독성 확보. 주황 화살표와는 색·형태가 모두 달라 혼동 없음.

MY가 다른 배지를 모두 가리는 이유?

내 매치는 이미 아는 정보. 거기에 다른 상태까지 겹치면 시각적으로 산만. MY 단독 노출로 클러터를 제거.

색이 많아지는 문제(레인보우)는?

색은 상태 신호(진행유력·신규·확정·긴급양도)에만 쓰고, 거래 정보(가격·MY)는 무채색으로 분리. "색=상태, 무채색=거래"로 색의 역할을 명확히 해 산만함을 통제.
